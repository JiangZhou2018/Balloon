# Balloon H5 战斗扩展规格

## 1. 目标
- 在现有 `自动开火 -> 打爆气球 -> 宝箱掉落` 主循环上，补充随机触发型武器模块。
- 新模块优先服务于两个目标：`更强爽点` 和 `更清晰构筑感`。
- 所有模块都必须能挂到现有局内升级系统，并允许以后通过页签分类查找。

## 2. 分类方案

### 2.1 战斗页升级页签
- `全部`：显示所有局内升级。
- `攻击`：基础火力、连射、暴击、穿透、弹跳、导弹、激光等输出模块。
- `防御`：保险上限、理赔减免、减速场等生存模块。
- `功能`：气球现金、坠箱金币、利息等经济模块。

### 2.2 攻击模块子分类
| 子类 | 模块 |
|------|------|
| 基础火力 | 伤害、射速、射程 |
| 弹幕特效 | 溅射、连射、暴击、穿透、弹跳 |
| 重武器 | 导弹、激光 |

## 3. 通用战斗结算规则

### 3.1 攻击周期
- `射速 fireRateMs` 仍然定义主武器开火周期。
- 每次主武器开火记为 `1 次主射击`。
- 连射、导弹、激光都从 `主射击` 触发，不单独改变主武器基础射速。

### 3.2 伤害公式
- 主子弹基础伤害：`baseDamage = runtimeStats.damage`
- 命中前护甲结算：`afterArmor = max(1, rawDamage - armor)`
- 暴击后公式：`rawDamage = baseDamage * critMultiplier`
- 溅射伤害：`splashDamage = round(afterArmor * splashRate + splashFlat)`
- 导弹和激光可直接使用独立系数，但最终仍需经过护甲结算。

### 3.3 触发顺序
1. 主射击锁定目标。
2. 生成 `shot` 事件。
3. 结算是否触发连射、导弹、激光。
4. 结算本发子弹是否暴击。
5. 命中主目标。
6. 若命中成功，按顺序处理：穿透 -> 弹跳 -> 溅射 -> 气球爆炸 -> 宝箱掉落。

### 3.4 概率统一规则
- 所有触发概率使用独立随机池，互不排斥。
- 单次主射击允许同时触发多个模块。
- 为避免场面失控，重武器额外加内部冷却：
  - 导弹内置冷却：`1.8 秒`
  - 激光内置冷却：`2.4 秒`

## 4. 模块规格

### 4.1 连射 Burst
| 项目 | 规格 |
|------|------|
| 触发条件 | 主射击发出时进行一次判定 |
| 初始值 | 0 |
| 参数 | `burstChance`、`burstCount`、`burstIntervalMs` |
| 建议范围 | 概率 `6%~35%`，次数 `1~4` 发额外子弹，间隔 `55~90ms` |
| 1~20级曲线 | 概率每级 `+1.2%`，5/10/15/20 级额外 `+1 发`，间隔随等级从 `90ms` 降到 `55ms` |
| 叠加规则 | 连射额外子弹继承当前伤害、暴击、穿透、弹跳；不重新触发导弹和激光 |
| 爽点目标 | 中期开始明显看到“一次开火扫掉一串气球” |

### 4.2 暴击 Crit
| 项目 | 规格 |
|------|------|
| 触发条件 | 每一发造成伤害的子弹单独判定 |
| 初始值 | 0 |
| 参数 | `critChance`、`critMultiplier` |
| 建议范围 | 概率 `5%~40%`，倍率 `1.6x~3.0x` |
| 1~20级曲线 | 概率每级 `+1.25%`，倍率每 4 级 `+0.1` |
| 叠加规则 | 暴击先于护甲结算；连射子弹可暴击；导弹和激光默认可独立暴击 |
| 爽点目标 | 爆出更大的数字、特效更亮，形成小高潮 |

### 4.3 穿透 Pierce
| 项目 | 规格 |
|------|------|
| 触发条件 | 子弹命中后判定是否继续穿过当前目标 |
| 初始值 | 0 |
| 参数 | `pierceChance`、`pierceCount`、`pierceDecay` |
| 建议范围 | 概率 `8%~45%`，次数 `1~3`，衰减 `100% -> 70% -> 50%` |
| 1~20级曲线 | 概率每级 `+1.1%`，7/14/20 级各 `+1 次` 穿透 |
| 叠加规则 | 优先穿同宝箱中的下一颗气球；若同宝箱已空，则可穿向同航道最近宝箱 |
| 爽点目标 | 形成“一枪串穿两三颗”的清理感 |

### 4.4 弹跳 Ricochet
| 项目 | 规格 |
|------|------|
| 触发条件 | 子弹命中后判定是否弹向附近目标 |
| 初始值 | 0 |
| 参数 | `ricochetChance`、`ricochetCount`、`ricochetRange`、`ricochetDecay` |
| 建议范围 | 概率 `10%~32%`，次数 `1~3`，范围 `18~28` 进度单位 |
| 1~20级曲线 | 概率每级 `+0.9%`，8/16 级各 `+1 次` 弹跳，范围每 5 级 `+2` |
| 叠加规则 | 弹跳优先选相邻航道最近宝箱；不可回弹到同一颗已命中的气球 |
| 爽点目标 | 多目标同屏时产生链式清怪感 |

### 4.5 导弹 Missile
| 项目 | 规格 |
|------|------|
| 触发条件 | 主射击发出时判定，且导弹内部冷却结束 |
| 初始值 | 0 |
| 参数 | `missileChance`、`missileDamageRate`、`missileRadius`、`missileSpeed` |
| 建议范围 | 概率 `4%~18%`，伤害系数 `2.2x~4.5x`，半径 `10~18` 进度单位 |
| 1~20级曲线 | 概率每级 `+0.45%`，伤害每级 `+0.12x`，半径每 5 级 `+2` |
| 叠加规则 | 导弹锁定当前主目标；爆炸范围内所有气球都受伤；导弹伤害可暴击但不触发连射 |
| 爽点目标 | 低频高爆发，一次清掉一束气球 |

### 4.6 激光 Laser
| 项目 | 规格 |
|------|------|
| 触发条件 | 主射击发出时判定，且激光内部冷却结束 |
| 初始值 | 0 |
| 参数 | `laserChance`、`laserCount`、`laserDurationMs`、`laserTickMs`、`laserDamageRate` |
| 建议范围 | 概率 `3%~16%`，数量 `1~3`，持续 `400~1100ms` |
| 1~20级曲线 | 概率每级 `+0.35%`，10/20 级各 `+1 条激光`，持续每级 `+35ms` |
| 叠加规则 | 激光每 `120ms` 造成一次伤害；单条激光优先锁当前目标，附加激光锁最近目标；激光可穿透同航道多个气球 |
| 爽点目标 | 形成明显的持续扫射和切割感 |

### 4.7 溅射 Splash
| 项目 | 规格 |
|------|------|
| 触发条件 | 主目标命中后稳定触发 |
| 初始值 | 当前已实现 |
| 参数 | `splashFlat`、`splashRate` |
| 建议范围 | 固定伤害 `1~18`，比例 `8%~35%` |
| 1~20级曲线 | 每级 `+1 固定伤害`，5/10/15/20 级额外 `+5% 比例` |
| 叠加规则 | 溅射不再二次触发溅射，但可触发气球爆炸与掉落 |

## 5. 升级曲线建议

### 5.1 成本曲线
- 所有新模块延用当前成本模型：`cost = round(baseCost * growth ^ level)`
- 建议基础成本：
  - 连射概率：`30`
  - 连射次数：`42`
  - 暴击率：`28`
  - 暴击伤害：`36`
  - 穿透概率：`34`
  - 穿透次数：`46`
  - 弹跳概率：`34`
  - 弹跳次数：`46`
  - 导弹概率：`52`
  - 导弹强化：`60`
  - 激光概率：`58`
  - 激光数量：`72`
- 建议成长系数：
  - 常规概率类：`1.34 ~ 1.4`
  - 次数类：`1.48 ~ 1.58`
  - 重武器类：`1.5 ~ 1.62`

### 5.2 20级分段
- `1~5级`：尽快给到首次触发体验，重点提升概率。
- `6~10级`：开始提升数量或范围。
- `11~15级`：强化持续时间、伤害系数、连锁次数。
- `16~20级`：少量提升概率，重点给节点奖励，如 `+1 激光`、`+1 连射`。

## 6. 与现有系统叠加规则

### 6.1 与伤害
- `damage` 仍是所有直接伤害模块的基础底数。
- 导弹和激光如果使用倍率伤害，基数统一取 `damage`。
- 例：`missileDamage = damage * missileDamageRate`

### 6.2 与射速
- `fireRate` 只影响主射击冷却。
- 连射属于一次主射击内部派生，不额外缩短主射速。
- 导弹与激光不占用主武器冷却，但受内部冷却限制。

### 6.3 与溅射
- 子弹命中后先处理主伤害，再处理溅射。
- 连射子弹也能触发溅射。
- 导弹自身带范围伤害，不再额外附带普通溅射，防止范围叠太夸张。
- 激光每次 tick 默认不触发普通溅射。

### 6.4 与护盾气球
- 护盾气球仍优先吸收锁定。
- 激光可持续压制护盾。
- 导弹爆炸优先对范围内护盾气球生效。

### 6.5 与集火
- 集火优先级高于随机目标搜索。
- 连射、导弹、激光在有集火目标时都默认优先锁定该目标。

## 7. 事件系统扩展

### 7.1 新增事件类型
| 事件 | 说明 | 关键字段 |
|------|------|------|
| `shot` | 主子弹发射 | `lane` `progress` `balloonIndex` `balloonCount` |
| `shotHit` | 子弹命中 | `targetChestId` `targetBalloonId` `damage` |
| `crit` | 暴击触发 | `multiplier` `damage` |
| `burstStart` | 连射开始 | `burstCount` `intervalMs` |
| `burstShot` | 连射子弹发射 | `sequence` |
| `pierce` | 穿透到下一个目标 | `remainingPierce` |
| `ricochet` | 弹跳到附近目标 | `remainingBounce` |
| `missileLaunch` | 导弹发射 | `targetChestId` |
| `missileExplode` | 导弹爆炸 | `radius` `damage` |
| `laserCast` | 激光施放 | `laserCount` `durationMs` |
| `laserTick` | 激光周期伤害 | `tickIndex` `damage` |
| `balloonPop` | 气球爆炸 | 现有字段可复用 |
| `chestDrop` | 宝箱掉落 | 现有字段可复用 |
| `cashGain` | 获得现金 | `amount` `source` |

### 7.2 事件使用建议
- `engine.ts` 继续负责推送标准化战斗事件。
- `Battlefield.tsx` 只消费事件做视觉表现，不直接参与数值计算。
- 事件保留 `id`，便于前端粒子与动画按帧消费。

## 8. UI 表现建议

### 8.1 升级列表
- 每张升级卡保留：`名称`、`等级`、`当前效果值`、`消耗现金`
- 去掉长描述，只在页签顶部或长按弹层里展示短说明。
- 新增模块后按页签筛选：
  - 攻击：伤害、射速、射程、溅射、连射、暴击、穿透、弹跳、导弹、激光
  - 防御：保险上限、理赔减免、减速场
  - 功能：气球现金、坠箱金币、利息

### 8.2 战斗表现
- 连射：子弹连发、枪口闪烁频率变高。
- 暴击：更亮的命中光圈和更大的爆字。
- 穿透：子弹穿过第一个气球后留下拖尾。
- 弹跳：子弹折线转向到下个目标。
- 导弹：明显大弹体、飞行尾烟、爆炸范围圈。
- 激光：持续光束、命中时闪烁切割线。

### 8.3 HUD 提示
- 触发重武器时，在战场中央短暂飘字：
  - `连射`
  - `暴击`
  - `导弹`
  - `激光`
- 飘字停留 `400~700ms`，不进入固定 HUD，避免界面过满。

## 9. 开发顺序建议
1. 先做 `暴击`、`连射`、`穿透`
2. 再做 `导弹`
3. 最后做 `激光` 和 `弹跳`

## 10. 最小可交付拆分

### 第一批
- 数据结构：新增升级定义和运行时属性
- 事件：`crit` `burstStart` `burstShot` `pierce`
- 表现：暴击闪光、连射子弹、穿透拖尾

### 第二批
- 数据结构：导弹属性和内部冷却
- 事件：`missileLaunch` `missileExplode`
- 表现：导弹飞行和爆炸粒子

### 第三批
- 数据结构：激光属性和 tick 伤害
- 事件：`laserCast` `laserTick` `ricochet`
- 表现：激光光束和折线弹跳
